# syslab

Учебная консольная программа, демонстрирующая работу с системными вызовами POSIX.
Каждая подкоманда намеренно реализована напрямую через POSIX-вызовы (без
буферизованного stdio для работы с файлами), чтобы наглядно показать поведение
конкретных syscalls.

## Требования

- Компилятор C с поддержкой C11 (`cc`, `gcc` или `clang`).
- POSIX-совместимая система. Код собирается с `-D_POSIX_C_SOURCE=200809L`
  и использует `fork`, `execvp`, `waitpid`, `lseek`, `stat`, `access`.

## Сборка

```sh
make           # собирает ./syslab
make clean     # удаляет бинарный файл
```

Переопределить компилятор/флаги можно через переменные окружения:

```sh
make CC=clang CFLAGS='-std=c11 -Wall -Wextra -O0 -g -D_POSIX_C_SOURCE=200809L'
```

## Использование

```
syslab <команда> [аргументы]
```

Без аргументов либо с неизвестной командой программа печатает справку в
stderr и завершается с кодом `1`.

### `info <путь>`

Выводит метаданные файла. Системные вызовы: `stat`, `access`.

```
$ ./syslab info file.txt
Path: file.txt
Type: regular file
Size: 1248 bytes
Readable: yes
Writable: yes
Executable: no
```

Поле `Type` принимает одно из значений: `regular file`, `directory`,
`symbolic link`, `character device`, `block device`, `FIFO`, `socket`,
`unknown`.

Особенности:

- Используется `stat`, а не `lstat`, поэтому символические ссылки
  раскрываются: `Type` показывает тип целевого объекта, а размер — размер
  цели. Чтобы увидеть саму ссылку, потребовалась бы доработка с `lstat`.
- Биты `Readable/Writable/Executable` проверяются через `access(2)` от лица
  реального UID/GID процесса (а не effective), поэтому результат соответствует
  правам текущего пользователя, а не возможной setuid-ситуации.
- Если `stat` завершился ошибкой (например, файл не существует), выводится
  `stat(<path>): <errno-сообщение>` в stderr и возвращается код `1`.

### `copy <исходный> <целевой>`

Копирует содержимое файла блоками по 8 КиБ. Системные вызовы: `open`, `fstat`,
`read`, `write`, `close`.

```
$ ./syslab copy a.txt b.txt
```

Особенности:

- Назначение открывается с `O_WRONLY | O_CREAT | O_TRUNC`, т.е. существующий
  файл будет **перезаписан**.
- Режим (биты прав) назначения берётся из исходного файла (`st_mode & 0777`),
  либо `0644`, если `fstat` исходного дескриптора не сработал.
- Копируется только содержимое: владелец, временные метки, расширенные
  атрибуты и ACL не сохраняются.
- При прерывании `write` сигналом (`EINTR`) запись повторяется. Прочие
  ошибки записи приводят к выходу с кодом `1`.
- Sparse-файлы копируются плотно (дыры превращаются в нули).

### `tail <файл> <N>`

Печатает последние `N` строк файла, читая блоками по 4 КиБ от конца к началу.
Системные вызовы: `open`, `lseek`, `read`, `write`, `close`.

```
$ ./syslab tail /var/log/syslog 20
```

Особенности:

- `N` должно быть положительным целым; иначе выводится ошибка и возвращается `1`.
- Завершающий перевод строки в конце файла не считается отдельной строкой
  (поведение совместимое с `tail(1)`).
- Если в файле меньше `N` строк, печатается весь файл.
- Работает только с тем, что поддерживает `lseek` (обычные файлы, не каналы
  и не сокеты).

### `run <программа> [аргументы...]`

Создаёт дочерний процесс через `fork`, запускает в нём указанную программу
через `execvp` и ожидает её завершения через `waitpid`.

```
$ ./syslab run ls -la /tmp
```

Коды возврата `run`:

| Ситуация                                  | Код возврата          |
|-------------------------------------------|-----------------------|
| Программа завершилась штатно              | её `WEXITSTATUS`      |
| Программу не удалось запустить (execvp)   | `127`                 |
| Программа убита сигналом                  | `128 + номер сигнала` |
| Ошибка `fork`/`waitpid`                   | `1`                   |

`execvp` ищет программу в `PATH`, если имя не содержит `/`.

## Коды возврата `syslab`

- `0` — успех (или код дочернего процесса для `run`).
- `1` — неверные аргументы, неизвестная команда или системная ошибка.
- Прочие коды — пробрасываются от `run` (см. таблицу выше).

## Типичные проблемы

- **`open(...): Permission denied`** — у текущего пользователя нет прав на
  файл. Проверьте `./syslab info <путь>`.
- **`tail`: ничего не выводится** — файл, скорее всего, пуст или состоит
  из одних пустых строк без переводов.
- **`run`: `execvp(...): No such file or directory`** — программа не
  найдена в `PATH`. Укажите полный путь или добавьте каталог в `PATH`.
- **Большой sparse-файл раздулся после `copy`** — ожидаемо: дыры
  материализуются в нулевые байты (см. выше).
